Running this CSS

This is 569 MB of pure CSS text.

You can open it in Chrome. It will work. The browser will evaluate every calc() expression, every if() condition, every var() reference — because that's what CSS engines do.

The catch: each CPU clock cycle requires Chrome to re-evaluate ~2 million calc() assignments across ~1 million @property declarations in a single animation frame. Chrome was never designed for this. It takes about 2 seconds per frame — giving us a clock speed of 0.5 Hz. The original 8086 ran at 5 MHz. This is ten million times slower.

Chrome (native CSS) ~0.5 Hz Zork prompt: ~4 months
Calcite (CSS JIT compiler) ~300 kHz Zork prompt: instant

Calcite is a JIT compiler for CSS — the same idea as V8 for JavaScript. It parses the exact same CSS file, recognises patterns (dispatch tables, broadcast writes, constant expressions), compiles them to bytecode, and executes them. It doesn't skip anything. It doesn't know what the CSS computes. It's just faster at evaluating it.

If Calcite disagrees with Chrome, Calcite is wrong.
