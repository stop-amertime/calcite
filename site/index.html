CSS-DOS

A DOS computer running in pure CSS.

No JavaScript. No WebAssembly. No cheating.
Every CPU register, every byte of memory, every instruction decode
is a CSS custom property evaluated by calc().

How it works
1.
A real x86 program (like Zork, a 101KB DOS executable from 1980) is transpiled into pure CSS.
2.
The CSS encodes the entire 8086 CPU: registers, flags, instruction decode, memory — all as @property declarations and calc() expressions. The result is 569 MB of CSS — 19 million lines.
3.
Open it in Chrome and it works. Slowly. The browser manages about one clock cycle every two seconds. At that rate, Zork printing “West of House” takes months.
4.
Calcite is a JIT compiler for CSS — like V8 is for JavaScript. It parses the exact same CSS, recognises patterns, and runs it at 300 kHz — 600,000× faster. No cheating.
